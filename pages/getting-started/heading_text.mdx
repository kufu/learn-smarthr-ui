まずは基本的なコンポーネントから学習を始めていきましょう。見出しや本文などのテキストは、画面を構成する上で最も小さく最も重要な要素です。`Heading`と`Text`の使い方を抑えていきましょう。

## Heading

`Heading`コンポーネントは、直後に続くコンテンツの見出しに使います。

次のコードを変更してタイトル名を変えてみましょう。要領としては、`h1`や`h2`といったHTMLの見出しタグの使い方と同じです。`<Heading>`と`</Heading>`の間のテキストを書き換えることで見出しを変更できます。

<SmartHRUIPreview
  files={{
    'App.tsx': `import { Heading } from 'smarthr-ui'

export default function App() {
  return (
    <Heading type="screenTitle" tag="h1">
      これはスクリーンタイトル
    </Heading>
  )
}`,
  }}
/>

Reactで作られたコンポーネントすべてに共通しますが、コンポーネントにはそれぞれ`Props`という、コンポーネントの振る舞いを変えるための設定があります。

`Heading`コンポーネントには文字サイズを変更する`type`と、出力要素（実際に描画するHTML要素）を変更する`tag`という`Props`が用意されています。
試しに`type`の値を変えてみましょう。`type`には`screenTitle`、`sectionTitle`、`blockTitle`、`subBlockTitle`、`subSubBlockTitle`の5つの値を指定できます。

<SmartHRUIPreview
  files={{
    'App.tsx': `import { Heading } from 'smarthr-ui'

export default function App() {
  return (
    <Heading type="pageTitle" tag="h1">
      これはページタイトル
    </Heading>
  )
}`,
  }}
/>

各`Props`の値は、予め決められた値の中から選ぶ必要があります。例えば、`type`に`title`という値を指定しても、その値は存在しないため正しく描画されません。

`Props`がどんな値を持つかは、デザインシステムの[HeadingコンポーネントのProps](https://smarthr.design/products/components/heading/#h2-4)を参照してください。

## Text

`Text`コンポーネントは段落や段落内のテキスト、ボタンのラベルなど、ありとあらゆつ文字の装飾に使います。デフォルトの出力要素は`span`で、文字サイズは`M`、行間は`NORMAL`、色は`inherit`です。これはSmartHRというプロダクトの特性上、段落や本文の割合が少ないことから「良いデフォルト」として決められています。

では試しに「p要素」で「文字サイズL」のTextを書いてみましょう。以下のコードに`Props`の属性から追加してみてください。

各`Props`がどんな値を持つかは、デザインシステムの[TextコンポーネントのProps](https://smarthr.design/products/components/text/#h2-0)を参照してください。

`as`という`Props`は存在しませんが、これはTextコンポーネントが内部的にstyled-componentsを使っているためです。[styled-componentsの`as`](https://styled-components.com/docs/api#as-polymorphic-prop)を参照してください。

<SmartHRUIPreview
  files={{
    'App.tsx': `import { Text } from 'smarthr-ui'

export default function App() {
  return (
    <Text>
      イカした本文が入ります。
    </Text>
  )
}`,
  }}
/>

変わりましたか？ `Heading`も`Text`も入れ子にできます。以下の例では`p`要素で出力した`Text`の中に`span`で出力した`Text`を挿入しています。

<SmartHRUIPreview
  files={{
    'App.tsx': `import { Text } from 'smarthr-ui'

export default function App() {
  return (
    <Text as="p">
      <Text color="TEXT_LINK" weight="bold">カラーや強調</Text>ができます。
    </Text>
  )
}`,
  }}
/>

注意点として、ReactもHTMLも要素の使い方が誤っていたとしてもエラーにはなりません。

例えば、`p`要素の中に`h1`要素を入れることはできますが、正しいHTMLではありません。たとえ見た目として問題なく表示されていたとしても、仕様や構造的に誤りがあれば、ブラウザを始めとした支援技術などの機械は正しく解釈できません。ウェブの上でプロダクトを作る作るは、その根幹となるHTMLの理解は必要になるでしょう。

筆者は[HTML Standard 日本語訳](https://momdo.github.io/html/)を読めるようになることをおすすめしています。
