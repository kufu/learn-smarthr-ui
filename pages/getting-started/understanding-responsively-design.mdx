ここでは UI としての画面を構成するために最低限必要な、*“レスポンシブなレイアウト”* について学びます。

## なぜ “レスポンシブデザイン” ではないのか

ウェブの文脈における “レスポンシブデザイン” と区別するために、あえて *“レスポンシブなレイアウト”* としています。

“レスポンシブデザイン” は様々な環境（デバイスや画面解像度、画面の向き、ブラウザに依る拡大、文字サイズなど）に対して、適切にレイアウトを変え、見やすく表示させようという考え方です。

利用者の動作環境が多様化し続ける現代において、そのすべてをコードで支配することは現実的に不可能です。人間はすべての状態を考えられるわけではありませんし、コンピューターがそれを補足するわけでもありません。

*“レスポンシブなレイアウト”* では、あらゆる環境で完璧に表示することは目指しません。モバイル端末は 375px 程度でタブレット端末は 768px 以上、デスクトップは 1024px 以上、と利用環境を定めに行くことは一定有効かもしれません。しかし、デスクトップ端末のブラウザで幅を 320px にしているとき、それは「モバイル端末」なんでしょうか。

利用環境や状態が利用者の手にある以上、ここで必要になるのは「どんな環境か」ではなく、「どう表示されてほしいか」です。そのためにも、文章やまとまり<ruby>毎<rp>（</rp><rt>ごと</rt><rp>）</rp></ruby>に、どんな振る舞いをするべきなのかを定義し、それを積み重ねていきましょう。

これは “[宣言的デザイン](https://yuheiy.com/2022-08-01-declarative-design)” に近しい考え方です。また、似た概念にアダプティブデザインがありますが、どちらもここでは取り上げません。

## レイアウトの最小単位は文字である

レイアウトにおける振る舞いを定義するための最小単位は、文字です。ピクセルではありません。

もしあなたがピクセルでレイアウトを考えていたとしても、それはデザインツールがピクセルで値を定義させるからです。あるいは表示される画面がピクセルであるためです。UI や画面レイアウトにおける振る舞いがピクセルを求めているわけではありません。

一般的なサイドメニューを想像してみましょう。そのサイドメニューは主要なコンテンツと並んでいます。そのサイドメニューの振る舞いはどう定義されるべきでしょうか? 様々な振る舞いの中からレイアウトに関わる「幅」について考えてみてください。

240px でしたか? あるいは 360px でしたか? いえ、きっとピクセルでは考えなかったでしょう。その「サイドメニュー」とは一体何のためのサイドメニューなのか、メニューの中にはどんな項目が並ぶのか、項目の文字量はどれくらいなのか、主要なコンテンツとは何でサイドメニューとの関係性は何なのか、などと考えたのではないでしょうか。

ウェブサイトであっても、ウェブアプリケーションであっても、あなたがウェブの上で何らかのコンテンツを提供する限り、そのコンテンツの主体は文字であることが多いでしょう[^1]。

あなたが作ろうとしている UI にはどんなコンテンツが表示されるのでしょう? そしてそれはどのように見せたいのでしょう? それを考えることがレイアウトを考え実装するための第一歩なのです。今あなたが読んでいる文章も、「1行は長すぎると読みづらくなるため、最大でも40文字にしよう」と考えて実装されています。

[^1] [ウェブデザインの95%はタイポグラフィ]()という言葉は、説明するのをはばかるほど有名です。

## SmartHR UI で提供しているレイアウトコンポーネントについて

SmartHR UI では、SmartHR を構築する上で最低限必要なコンポーネントを「レイアウトコンポーネント」と称して提供しています。

コンポーネントは組み合わせて利用されるため、コンポーネント自身がどこでどのように使われるかを知ることはできません。そのためコンポーネントは自身の周囲に余白（`margin`）を持ちません。しかし、使いやすい画面を提供するためには何らかの形でその「余白」を実装しなければなりません。

そこでレイアウトコンポーネントを使用します。レイアウトコンポーネントの多くは「余白を管理するコンポーネントである」といっても過言ではありません。

つまり、このレイアウトコンポーネントの振る舞いを理解し、適切に扱えるようになれば、余白を思うままに操り、使いやすい UI としての画面を実装できるのです。

また、SmartHR UI のレイアウトコンポーネントは基本的に [Every Layout]() を参考に実装されています。[日本語の翻訳書]()も発売されているため、一読をおすすめします。
